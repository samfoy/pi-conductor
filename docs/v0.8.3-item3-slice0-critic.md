## 3. Plan deviation in `1e4f035` (`formatToolCallShort` untouched)

### Builder's claim verified against source

`src/event-handler.ts:110-130` (`formatToolCallShort`):

| Branch | What it does | Conflict with `summarizeToolArgs` (core) |
|---|---|---|
| `bash` | `$ ${cmd[:50]}…` (50-char head before ellipsis) | Core trims at 49+`…` (`MAX_SUMMARY_LEN=50` = 49 chars + ellipsis). **Different head budget.** |
| `read`/`write`/`edit` | `read ${shortenPath(file_path)}` (HOME→`~` substitution; no length trim) | Core has no `shortenPath`; trims at 50. **Different shortening.** |
| `grep` | `grep ${pattern}` (verbatim, **no truncation**) | Core trims at 50. **Different policy.** |
| default | bare `name` | Core emits `key=value` pairs. **Different shape entirely.** |

Builder's claim — "no branch is conflict-free for delegation" — is **factually correct**. Plan §A2's "thin wrapper" prescription assumed delegation was possible; on the actual code it isn't without changing widget output (`run.lastToolCall` is read by `widget.formatRow`). The plan was over-optimistic; builder's call to leave `formatToolCallShort` untouched + pin the contract via byte-exact tests is the correct call.

### Was this a dodge?

No. The plan's own acceptance criterion #5 ("`event-handler.ts:84` … emits the same string shape as before — byte-identical") is **stronger than its prescription** ("thin wrapper"). The byte-exact requirement wins; the wrapper was a means, not the goal. Builder honored the goal and pinned the means with regression tests.

### Regression test coverage (9 new cases at `tests/event-handler.test.ts:332-381`)

| Plan-required scenario | Test name | Pinned? |
|---|---|---|
| bash short | "bash short command keeps `$ ` prefix and command literal" | ✅ |

10 cases (one combines bash, read, grep undefined-args). All plan scenarios covered.

### Forward-looking concern (Slice 1)

Slice 1 lands `shortenMiddle` for bash arg tails in `src/tool-summary.ts`. **The widget's `lastToolCall` (via `formatToolCallShort`) is the OTHER place that renders truncated bash commands.** v0.8.1 sub-issue #1 reads: *"truncation cuts at the wrong place — `bash aws sts ... 2>/…` and `bash ada credentials ... --account=...` hide the meaningful argument tail."* That complaint is rendered through both:

- transcript (collapsed `▸ bash …`) — via `summarizeToolArgs` → fixed by Slice 1.
- widget row (`run.lastToolCall` = `$ ada credentials …`) — via `formatToolCallShort` → **not fixed by Slice 1** as currently scoped.

If sub-issue #1 is supposed to fix the widget too, Slice 1 is incomplete by design.
The plan needs to clarify (see §6).

## 6. Recommendation for Slice 1

**Slice 1 may proceed**, but the plan needs a one-paragraph amendment first.

### Amendment needed

The plan currently scopes Slice 1 to `src/tool-summary.ts` only. Sub-issue #1 ("truncation cuts at the wrong place") is rendered through **both** the transcript renderer AND the widget's `lastToolCall` row. Slice 1 as written fixes the transcript half; the widget half (via `formatToolCallShort` in `event-handler.ts`) keeps its head+ellipsis truncation.

Three options for the planner / user:

- **(a)** Slice 1 as-written. `shortenMiddle` lands in `tool-summary.ts`; transcript fixed; widget row keeps its current behavior. **Sub-issue #1 partially closed** (transcript only). Cheapest path; widget polish deferred.
- **(b)** Slice 1 expanded. `shortenMiddle` lands in `tool-summary.ts` AND `event-handler.ts`'s `formatToolCallShort` switches its bash branch from head-truncate to `shortenMiddle`. Touches one more file + 1–2 byte-exact regression tests get rewritten (the bash long-command case). **Sub-issue #1 fully closed.** Modest extra scope.
- **(c)** Defer the widget half. Slice 1 stays as-written; add Slice 1.5 (or a v0.9 ticket) for the widget half. **Sub-issue #1 closed in two passes.**

Recommendation: **(b)**. The cost is minimal (one branch swap + rewriting one regression test from `"$ " + "a".repeat(50) + "…"` to a `shortenMiddle`-shaped expectation), and it closes the user-facing complaint completely in one commit.

### Followup tickets (do not block Slice 1)

- **F1.** STATUS_GLYPH dedup leak — `src/history.ts:36` (full table) and `src/commands.ts:349` (switch statement) still hold their own copies. Trivial cleanup, ≤30 LoC + 1 test update. Could fold into Slice 0 as a fast-follow commit, or land standalone before Slice 1. Recommend fast-follow now; the followup is small and the dedup goal is in-flight.
- **F2.** Plan §A2 prescription ("thin wrapper for `formatToolCallShort`") was contradicted by reality. Update plan.md to reflect the as-built decision (regression-test-pinned divergence).
