#### Goal

Ship a pure function `resolveOnCompleteHook(perCall, cfg, persona)` in a
new `src/hook-cascade.ts` that resolves the 5-layer cascade
(per-call > project > user > persona-frontmatter > none) with

#### Acceptance criteria

- New file `src/hook-cascade.ts` exports `resolveOnCompleteHook`.
- Signature: `(perCall: HookSpec | undefined, cfg: ConductorConfig, persona: Persona) ‚Üí ResolvedHook | undefined`.

- Timeout is resolved from the same layer the command came from; falls back to default `300` seconds when unset.
- The function is pure: no `fs`, no `child_process`, no `Date.now()`. Verifiable by `grep -E 'fs|child_process|process\.|Date\.' src/hook-cascade.ts` returning empty.
- Inline doc comment on the function explains R7 (cascade explicit-disable surprise per call doesn't persist) per design ¬ß8.

  `"user"` distinction. The signature-deviation rationale is pinned
  inline at `src/hook-cascade.ts:38‚Äď48`.

  handler), extract project and user `HookSpec` candidates from the
  raw config files (or a new accessor exposed on
